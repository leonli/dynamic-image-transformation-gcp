Use the solution

The primary request style is a base64-encoded JSON document in the URL path (a Default request). Thumbor-style paths are covered in Thumbor compatibility; the full schema and error tables are in the API reference.

Build a Default request

Construct a JSON document, base64-encode it, and append it to your endpoint:

{
  "bucket": "my-image-bucket",
  "key": "photos/cat.jpg",
  "edits": {
    "resize": { "width": 300, "height": 300, "fit": "cover" }
  },
  "outputFormat": "webp"
}

  • edits keys are applied in order after resize; most sharp operations pass through directly (see the schema).
  • With no edits and no outputFormat, the original object is returned unmodified.

Node.js — build and sign a URL

const crypto = require("crypto");

const SECRET = process.env.DIT_SECRET;      // value stored in Secret Manager under SECRET_KEY
const ENDPOINT = "https://img.example.com";

function buildUrl(request, query = {}) {
  const path = "/" + Buffer.from(JSON.stringify(request)).toString("base64");

  // string-to-sign = path + ("?" + query sorted by key, signature excluded)
  const qs = Object.keys(query).sort()
    .map((k) => `${k}=${query[k]}`).join("&");
  const stringToSign = qs ? `${path}?${qs}` : path;

  const signature = crypto.createHmac("sha256", SECRET)
    .update(stringToSign).digest("hex");

  const sep = qs ? "&" : "?";
  return `${ENDPOINT}${path}${qs ? "?" + qs : ""}${sep}signature=${signature}`;
}

console.log(buildUrl(
  { bucket: "my-image-bucket", key: "photos/cat.jpg",
    edits: { resize: { width: 300, fit: "cover" } } },
  { expires: "20261231T235959Z" }
));

Python — build and sign a URL

import base64, hashlib, hmac, json, os

SECRET = os.environ["DIT_SECRET"].encode()
ENDPOINT = "https://img.example.com"

def build_url(request: dict, query: dict | None = None) -> str:
    path = "/" + base64.b64encode(
        json.dumps(request, separators=(",", ":")).encode()).decode()

    qs = "&".join(f"{k}={v}" for k, v in sorted((query or {}).items()))
    string_to_sign = f"{path}?{qs}" if qs else path

    signature = hmac.new(SECRET, string_to_sign.encode(), hashlib.sha256).hexdigest()
    sep = "&" if qs else "?"
    return f"{ENDPOINT}{path}{'?' + qs if qs else ''}{sep}signature={signature}"

print(build_url(
    {"bucket": "my-image-bucket", "key": "photos/cat.jpg",
     "edits": {"resize": {"width": 300, "fit": "cover"}}},
    {"expires": "20261231T235959Z"},
))

Signing is only enforced when the deployment sets ENABLE_SIGNATURE=Yes. A missing signature returns 400 AuthorizationQueryParametersError; a wrong one returns 403 SignatureDoesNotMatch. Signatures produced by clients written for the AWS solution verify unchanged.

Query parameter edits

These parameters overlay (and override) the edits from the path, for any request type:

https://img.example.com/<base64>?width=400&height=300&fit=cover&format=webp&grayscale=true

Supported: format, fit, width, height, rotate, flip, flop, grayscale / greyscale. A value of 0 or an empty string clears width/height; "0", "false" and "" are falsy for flip/flop/grayscale. Any other query parameter is dropped at the edge (it never varies the cache key). Parse failures return 400 QueryParameterParsingError.

Common edits

Resize, format, quality

{
  "key": "photos/cat.jpg",
  "edits": {
    "resize": { "width": 800, "fit": "inside" },
    "jpeg": { "quality": 75 }
  },
  "outputFormat": "jpeg"
}

fit accepts cover, contain, fill, inside (default), outside. Width/height are rounded; values ≤ 0 return 400 InvalidResizeException. For WebP output you can add a top-level "effort": 0–6 (default 4).

Smart crop (face detection)

{ "key": "people/team.jpg",
  "edits": { "smartCrop": { "faceIndex": 0, "padding": 40 } } }

Cloud Vision FACE_DETECTION finds faces; the crop is the selected face’s bounding box expanded by padding pixels and clamped to the image. Faces are ordered by bounding-box area, largest first, so faceIndex: 0 is always the most prominent face. If no face is found the full image is returned. Out-of-range values return 400 SmartCrop::FaceIndexOutOfRange or 400 SmartCrop::PaddingOutOfBounds. "smartCrop": true is shorthand for the defaults.

Content moderation

{ "key": "ugc/upload-123.jpg",
  "edits": { "contentModeration": {
      "minConfidence": 75, "blur": 50,
      "moderationLabels": ["Adult", "Violence"] } } }

Cloud Vision SafeSearch likelihoods are mapped to a 0–100 confidence (VERY_UNLIKELY=0 … VERY_LIKELY=100) and compared with minConfidence (default 75). If any selected label matches — categories Adult, Violence, Racy, Medical, Spoof; Rekognition aliases like "Explicit Nudity" and "Suggestive" are also accepted — the image is blurred with the given sigma (effective range 0.3–1000). Omit moderationLabels to match any category.

Overlay (watermark)

{ "key": "photos/cat.jpg",
  "edits": {
    "resize": { "width": 800 },
    "overlayWith": {
      "bucket": "my-image-bucket",
      "key": "brand/logo.png",
      "alpha": "50",
      "wRatio": "20",
      "options": { "left": "-10", "top": "90p" }
    } } }
  • alpha — extra transparency 0–100 (out-of-range values are treated as 0).
  • wRatio/hRatio — scale the overlay to a percentage of the base image.
  • options.left/top — pixel offsets; negative values measure from the right/bottom edge; NNp means percent (e.g. 90p = 90% down).
  • A failed overlay returns 400 OverlayImageException.

Round crop

{ "key": "avatars/user.jpg",
  "edits": { "resize": { "width": 200, "height": 200, "fit": "cover" },
             "roundCrop": true } }

Cuts an ellipse (default: the inscribed circle/ellipse) and trims the result. You can pass an object with rx, ry, top, left to control the ellipse. Output formats without alpha (JPEG) render the outside black — combine with "outputFormat": "png" or webp to keep transparency.

Animated GIF / WebP

{ "key": "stickers/dance.gif",
  "edits": { "resize": { "width": 240 } },
  "outputFormat": "webp" }

GIF sources are processed frame-by-frame automatically (or force behavior with "edits": { "animated": true|false }). Single-page images fall back to static processing. For animated images, rotate, smartCrop, roundCrop and contentModeration are skipped.

Custom response headers

{ "key": "docs/diagram.png",
  "headers": { "Cache-Control": "max-age=86400,public",
               "X-Robots-Tag": "noindex" } }

Headers are filtered through a deny list (security-sensitive names such as Set-Cookie, Authorization and x-amz-*/ content-* prefixes are dropped) — see the API reference. Only Default requests support custom headers.

Expiring URLs

https://img.example.com/<base64>?expires=20261231T235959Z&signature=…

The timestamp is strict UTC YYYYMMDDTHHmmssZ. Malformed values return 400 ImageRequestExpiryFormat; past timestamps return 400 ImageRequestExpired. On success the response Cache-Control is rewritten to max-age=<seconds-left>,public so the CDN never serves beyond the expiry. Combine with signatures — otherwise the parameter can simply be removed by the caller.

Fallback image

With ENABLE_DEFAULT_FALLBACK_IMAGE=Yes and DEFAULT_FALLBACK_IMAGE_BUCKET/KEY configured, any processing error returns the fallback image instead of an error body — the HTTP status code still reflects the original error, so monitoring keeps working. Cache-Control precedence: the fallback object’s own metadata → request headers → max-age=31536000,public.

Automatic WebP

With AUTO_WEBP=Yes, requests whose Accept header includes image/webp get WebP output. Precedence: edits.toFormat > AUTO_WEBP > JSON outputFormat. The handler emits Vary: Accept, which Cloud CDN honors, so both variants cache correctly.
