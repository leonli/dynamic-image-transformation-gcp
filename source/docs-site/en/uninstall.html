Uninstall the solution

Use the method that matches how you deployed.

Launch Wizard deployments

./infra/launch-wizard/launch-wizard.sh --destroy

The wizard replays the recorded variable values, shows the destruction plan and asks for confirmation before running terraform destroy under the hood.

Terraform deployments

cd infra/terraform
terraform destroy

Review the plan carefully before confirming.

What is removed

  • Cloud Run service dit-image-handler and its serverless NEG
  • Load balancer dit-lb: forwarding rule, global IP, managed SSL certificate, URL map, backend services and Cloud CDN configuration
  • Solution buckets (…-dit-demo, …-dit-docs, …-dit-logs) and their contents
  • Secret dit-signature-secret (if created by the deployment)
  • Resource-level IAM bindings created by the deployment

What is kept

  • Your source image buckets — the deployment never creates or deletes customer data buckets listed in source_buckets.
  • Container images in Artifact Registry (repository dit) — delete manually if unwanted: 
    gcloud artifacts repositories delete dit --location=asia-southeast1
  • Exported logs already stored in Cloud Logging (subject to retention).
  • Any DNS records you created — remove the A record for the image domain yourself.

Before destroying: if clients still resolve your image domain to this load balancer, they will start failing immediately. Repoint or remove the DNS record first and let the TTL expire.

If a destroy is interrupted, re-run it — Terraform resumes from recorded state. As a last resort, verify in the console that the resources above (search prefix dit-) are gone.
