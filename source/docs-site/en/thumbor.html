Thumbor compatibility

Thumbor-style paths work exactly as in the AWS solution — this page is the key reference for teams migrating existing CloudFront image URLs.

Path anatomy

/[fit-in/][AxB:CxD/][WxH/][filters:f1(v):f2(v)/]key
SegmentMeaningExample
fit-inResize with fit=inside (letterbox instead of crop)./fit-in/300x300/cat.jpg
AxB:CxDManual crop: left=A, top=B, width=C−A, height=D−B (each 1–6 digits)./10x20:110x220/cat.jpg
WxHResize target. A dimension of 0 means “auto” and implies fit=inside./300x0/cat.jpg
filters:…Colon-separated filter list. Filters are sorted alphabetically before they are applied (so format always precedes quality)./filters:format(webp):quality(80)/cat.jpg
keyObject key in the default source bucket, URL-decoded.photos/cat.jpg

Bucket prefixes: s3: and gs:

A path segment of the form s3:<bucket>/ or gs:<bucket>/ selects a different source bucket. If the bucket is on the SOURCE_BUCKETS allow list it overrides the default; otherwise the segment is ignored.

/300x300/gs:my-other-bucket/photos/cat.jpg
/300x300/s3:legacy-aws-bucket/photos/cat.jpg   ← still accepted after migration

BUCKET_MAP: migrate S3 URLs without rewriting them

If your existing URLs embed S3 bucket names that don’t exist on GCS, configure BUCKET_MAP with comma-separated s3name=gcsname pairs:

BUCKET_MAP="prod-images-us-east-1=my-gcs-images,assets-bucket=my-gcs-assets"

Then /300x300/s3:prod-images-us-east-1/cat.jpg transparently reads gs://my-gcs-images/cat.jpg. Allow-listing applies to the mapped (GCS) name. Combined with the unchanged signature algorithm, most AWS clients migrate with a DNS change only.

Custom rewrites

If your legacy URLs don’t match Thumbor syntax, set REWRITE_MATCH_PATTERN (a regular expression) and REWRITE_SUBSTITUTION. Matching paths are rewritten first and then parsed as Thumbor paths — the same Custom request mechanism as the AWS solution.

REWRITE_MATCH_PATTERN="/thumbs/(\d+)/(.*)"
REWRITE_SUBSTITUTION="/fit-in/$1x$1/$2"
# /thumbs/200/cat.jpg  →  /fit-in/200x200/cat.jpg

Migration checklist (CloudFront → Cloud CDN)

  1. Deploy with SOURCE_BUCKETS pointing at your GCS buckets; add BUCKET_MAP entries for any S3 bucket names embedded in URLs.
  2. Copy the signature secret JSON into Secret Manager unchanged and keep the same SECRET_KEY — existing signed URLs verify as-is.
  3. Keep REWRITE_MATCH_PATTERN/REWRITE_SUBSTITUTION values from your AWS deployment.
  4. Point the image domain’s DNS at the load balancer IP (test first with a staging hostname).
  5. Run deployment/run-e2e-tests.sh against the new endpoint and compare a sample of production URLs byte-for-byte or visually.
