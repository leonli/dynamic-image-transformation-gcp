Cost

Estimates only. The figures below are rough estimates based on public list prices for the asia-southeast1 region as of July 2026 and the stated assumptions. Your actual bill depends on image sizes, cache hit ratio, traffic geography and negotiated discounts. Always verify with the Google Cloud pricing calculator.

Assumptions

  • Average transformed image size: 45 KB
  • Cloud CDN cache hit ratio: 90% (only misses reach Cloud Run)
  • Cloud CDN egress to APAC internet: $0.09–0.14 / GB (tier dependent)
  • Cloud Run: 1 vCPU / 1 GiB, ~250 ms average processing per cache miss, min-instances=0
  • Load balancer: 1 forwarding rule ≈ $18/month, plus outbound data processing ≈ $0.008–0.012/GB
  • Source image storage is excluded (customer-owned buckets)

Optional: Cloud Vision API. smartCrop and contentModeration each issue one Vision call (FACE_DETECTION / SAFE_SEARCH_DETECTION) per cache miss, at ≈ $1.50 per 1,000 images after the monthly free tier. Example: if 5% of your 10M-tier misses use smart crop, that is 50,000 calls ≈ $75/month. Because results are cached by the CDN, Vision cost scales with misses, not with total views.

Cost levers. Raise the cache hit ratio (long Cache-Control, stable URLs, normalized query strings — the solution does this for you), enable AUTO_WEBP to cut egress bytes by 25–35%, and keep min-instances=0 unless cold-start latency matters.

Security

Request signing

Set ENABLE_SIGNATURE=Yes to require an HMAC-SHA256 signature on every URL. The secret lives in Secret Manager as a JSON document; SECRET_KEY selects the JSON key. The string-to-sign is the path plus the sorted query string, and the algorithm is identical to the AWS solution, so signatures generated by existing AWS clients keep working. See Use the solution — signed requests.

Source bucket allow list

SOURCE_BUCKETS is a comma-separated allow list; the first entry is the default bucket. Requests naming any other bucket are rejected with 403 ImageBucket::CannotAccessBucket. Grant the Cloud Run runtime service account roles/storage.objectViewer on these buckets only — the service never needs write access.

Network ingress

The Cloud Run service uses ingress=internal-and-cloud-load-balancing, so it can only be reached through the load balancer (and therefore through Cloud CDN). Direct calls to the *.run.app URL are blocked.

Other controls

  • Header deny list — custom response headers in Default requests are filtered against a deny list (authorization, set-cookie, x-amz-* prefixes, …); see the API reference.
  • CORS — off by default; enable with CORS_ENABLED=Yes and set CORS_ORIGIN.
  • Expiring URLs — the expires query parameter rejects stale links with 400 ImageRequestExpired.
  • Demo UI — not intended for production use; deploy it only in evaluation environments. Signature-enabled deployments are not supported by the demo.

Region choice

The reference deployment uses asia-southeast1 (Singapore) for Cloud Run and the buckets. Cloud CDN and the load balancer are global regardless of the backend region; pick the region closest to your origin data and to the majority of cache-miss traffic.
