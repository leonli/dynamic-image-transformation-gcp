Solution overview

Dynamic Image Transformation for Google Cloud CDN is a serverless image processing solution that resizes, converts, watermarks and optimizes images on the fly, directly from URLs — with results cached at the edge by Cloud CDN. It is a Google Cloud counterpart of the AWS solution Dynamic Image Transformation for Amazon CloudFront, designed so that customers migrating from AWS can keep their existing image URLs, request schema, Thumbor filters, signature algorithm and error contracts unchanged.

Migrating from AWS? URL formats (base64 JSON and Thumbor paths), the HMAC-SHA256 signature, all error codes and even the s3:<bucket>/ path prefix continue to work. See Thumbor compatibility for the migration mapping features.

Features

On-the-fly transformation

Resize, crop, rotate, flip, blur, sharpen, format conversion (JPEG/PNG/WebP/AVIF/GIF/TIFF) and quality control through the sharp pipeline.

Three request styles

Base64-encoded JSON requests, Thumbor-compatible paths with 20 filters, and custom rewrite rules — identical semantics to the AWS solution.

Smart crop & moderation

Face-aware cropping and SafeSearch-based content moderation powered by Cloud Vision API (counterpart of Amazon Rekognition).

Edge caching

Cloud CDN in front of a global external Application Load Balancer caches transformed images worldwide; the cache key includes the Accept header for automatic WebP.

URL signing

Optional HMAC-SHA256 request signatures with the key stored in Secret Manager — byte-for-byte compatible with AWS deployments.

Resilience

Optional fallback image on processing errors, bucket allow-listing, header deny list, and AWS-compatible error JSON for every failure mode.

Use cases

  • Responsive web delivery — generate the exact size and format each device needs from a single master image, with automatic WebP for supporting browsers.
  • E-commerce catalogs — thumbnails, zoom views and watermarked product shots from one source object, cached at the edge.
  • User-generated content — moderate uploads with SafeSearch and blur flagged content before it is served.
  • AWS migration — lift image workloads from CloudFront + Lambda to Cloud CDN + Cloud Run without touching client URLs.
