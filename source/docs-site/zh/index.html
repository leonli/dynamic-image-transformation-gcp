方案概览

Dynamic Image Transformation for Google Cloud CDN 是一套 serverless 图片实时处理方案:通过 URL 即可完成缩放、格式转换、加水印等操作,处理结果由 Cloud CDN 缓存在全球边缘节点。本方案是 AWS 解决方案 Dynamic Image Transformation for Amazon CloudFront 的 Google Cloud 对标实现——URL 格式、请求 JSON schema、Thumbor filter、签名算法、错误码逐一兼容,从 AWS 迁移的客户可以原封不动地沿用现有图片 URL。

正在从 AWS 迁移?base64 JSON 与 Thumbor 两种 URL 形式、HMAC-SHA256 签名、全部错误码,甚至 s3:<bucket>/ 路径前缀都继续可用。迁移映射能力详见 Thumbor 兼容。

核心能力

实时图片变换

基于 sharp 管线的缩放、裁剪、旋转、翻转、模糊、锐化、格式转换(JPEG/PNG/WebP/AVIF/GIF/TIFF)与质量控制。

三种请求形式

base64 编码的 JSON 请求、含 20 个 filter 的 Thumbor 兼容路径、自定义重写规则——语义与 AWS 方案完全一致。

智能裁剪与内容审核

由 Cloud Vision API(对标 Amazon Rekognition)提供人脸感知裁剪(smart crop)与基于 SafeSearch 的内容审核。

边缘缓存

全球外部应用负载均衡器(Application Load Balancer)前置 Cloud CDN,处理结果全球缓存;自动 WebP 的不同变体通过 Vary: Accept 响应头分开缓存。

URL 签名

可选的 HMAC-SHA256 请求签名,密钥存于 Secret Manager——与 AWS 部署逐字节兼容。

稳健性

处理失败时可返回兜底图(fallback image),源桶白名单、响应头拒绝列表(deny list),错误 JSON 与 AWS 逐字段一致。

典型场景

  • 响应式 Web 分发 —— 一张母图按设备实时生成所需尺寸与格式,支持 WebP 的浏览器自动降流量。
  • 电商图库 —— 缩略图、放大图、带水印商品图全部来自同一源对象,并缓存在边缘。
  • 用户生成内容(UGC) —— 用 SafeSearch 审核上传内容,命中即模糊处理后再对外提供。
  • AWS 迁移 —— 图片业务从 CloudFront + Lambda 平移到 Cloud CDN + Cloud Run,客户端 URL 一行不改。
