故障排查

403 SignatureDoesNotMatch

服务端算出的 HMAC 与 ?signature= 不一致。按此清单核对:

  • 待签串(string-to-sign)必须是含前导斜杠的 path,加 ? 与按键排序后的其余 query 参数(signature 本身除外),中间不得再做 URL 编码。
  • JSON 字节级一致。签名覆盖的是 base64 字符串,任何 JSON 重新序列化(键序、空白、separators)都会得到不同的 path。要对实际发送的字符串签名。
  • 密钥取值。Secret Manager 里存的是 JSON 文档,HMAC 密钥是 SECRET_KEY 指向的值,不是整份 JSON。核对: 
    gcloud secrets versions access latest --secret=dit-signature-secret
  • 摘要格式是小写十六进制(hex),不是 base64。
  • 没带签名(而非签错)返回 400 AuthorizationQueryParametersError;出现 500 SignatureValidationFailure 通常是服务读不到密钥——确认运行时服务账号有 roles/secretmanager.secretAccessor。

404 NoSuchKey

“The image <key> does not exist or the request may not be base64 encoded properly.” 任何 GCS 读取失败都会映射到这个错误,按顺序排查:

  1. 解码自己的 URL,检查 JSON 内容: 
    echo 'eyJrZXkiOiJwaG90b3MvY2F0LmpwZyJ9' | base64 -d
    base64 被截断或被中间系统改写(结尾 = 被去掉、+ 被换成空格)是最常见原因。
  2. 对象是否存在?gsutil ls gs://my-image-bucket/photos/cat.jpg —— key 区分大小写,且不能以斜杠开头。
  3. 权限。Cloud Run 运行时服务账号需要源桶上的 roles/storage.objectViewer。权限不足的读取同样表现为 404(设计如此,避免泄露桶结构)。
  4. 桶用错了。省略 bucket 时取 SOURCE_BUCKETS 的第一个——确认它确实是你预期的桶。

CDN 缓存不命中

先看响应头:

curl -sI "https://img.example.com/<base64>" | grep -i -e age -e cache-control -e etag
  • 有 Age 且在增长 → 响应来自 CDN 边缘缓存;没有 Age → 未命中(或响应不可缓存)。
  • 检查 Cache-Control。Cloud CDN 遵循源站响应头。若源对象元数据带 Cache-Control: private 或 no-store,会被透传,CDN 不缓存——修正对象元数据(gsutil setmeta -h "Cache-Control:max-age=31536000,public" gs://…),或清掉它以启用默认的 max-age=31536000,public。
  • expires URL 会把 Cache-Control 改写为剩余有效期——短时效链接缓存效果差是设计使然。
  • URL 不稳定。缓存键 = 完整 URL + 归一化后的 Accept 头。随机 query 参数会被归一化剔除(不会打散缓存),但路径不同——例如 JSON 键序不固定——会各自成为独立缓存项。请确定性地序列化 JSON。
  • 部署后的第一次请求必然未命中;错误响应只缓存 10 秒(4xx)/ 600 秒(5xx)。

Vision 相关错误(smartCrop / contentModeration)

  • 500 SmartCrop::Error 或 500 Rekognition::DetectModerationLabelsError —— Vision API 调用失败。确认已启用 API(gcloud services enable vision.googleapis.com)、运行时服务账号有调用权限,再看配额(默认约 1,800 次/分钟)。
  • 400 SmartCrop::FaceIndexOutOfRange —— 检出的人脸数少于 faceIndex。注意人脸按包围框面积从大到小排序,多人脸图片上可能与 Rekognition 的顺序不同。
  • 审核“不起作用” —— 可能性等级到置信度的映射是粗粒度的(步长 25);minConfidence 为 75 意味着至少 LIKELY 才命中。想连 POSSIBLE 一起拦,调到 50。
