使用方案

主推的请求形式是把 base64 编码的 JSON 文档放进 URL 路径(即 Default 请求)。Thumbor 风格路径见 Thumbor 兼容;完整 schema 与错误码表见 API 参考。

构造 Default 请求

先组装 JSON,再 base64 编码,拼到端点后面:

{
  "bucket": "my-image-bucket",
  "key": "photos/cat.jpg",
  "edits": {
    "resize": { "width": 300, "height": 300, "fit": "cover" }
  },
  "outputFormat": "webp"
}

  • edits 在 resize 之后按键序依次执行;多数 sharp 操作直通(见 schema)。

Node.js —— 构造并签名 URL

const crypto = require("crypto");

const SECRET = process.env.DIT_SECRET;      // value stored in Secret Manager under SECRET_KEY
const ENDPOINT = "https://img.example.com";

function buildUrl(request, query = {}) {
  const path = "/" + Buffer.from(JSON.stringify(request)).toString("base64");

  // string-to-sign = path + ("?" + query sorted by key, signature excluded)
  const qs = Object.keys(query).sort()
    .map((k) => `${k}=${query[k]}`).join("&");
  const stringToSign = qs ? `${path}?${qs}` : path;

  const signature = crypto.createHmac("sha256", SECRET)
    .update(stringToSign).digest("hex");

  const sep = qs ? "&" : "?";
  return `${ENDPOINT}${path}${qs ? "?" + qs : ""}${sep}signature=${signature}`;
}

console.log(buildUrl(
  { bucket: "my-image-bucket", key: "photos/cat.jpg",
    edits: { resize: { width: 300, fit: "cover" } } },
  { expires: "20261231T235959Z" }
));

Python —— 构造并签名 URL

import base64, hashlib, hmac, json, os

SECRET = os.environ["DIT_SECRET"].encode()
ENDPOINT = "https://img.example.com"

def build_url(request: dict, query: dict | None = None) -> str:
    path = "/" + base64.b64encode(
        json.dumps(request, separators=(",", ":")).encode()).decode()

    qs = "&".join(f"{k}={v}" for k, v in sorted((query or {}).items()))
    string_to_sign = f"{path}?{qs}" if qs else path

    signature = hmac.new(SECRET, string_to_sign.encode(), hashlib.sha256).hexdigest()
    sep = "&" if qs else "?"
    return f"{ENDPOINT}{path}{'?' + qs if qs else ''}{sep}signature={signature}"

print(build_url(
    {"bucket": "my-image-bucket", "key": "photos/cat.jpg",
     "edits": {"resize": {"width": 300, "fit": "cover"}}},
    {"expires": "20261231T235959Z"},
))

只有部署设置了 ENABLE_SIGNATURE=Yes 才会强制校验签名。缺签名返回 400 AuthorizationQueryParametersError,签名错误返回 403 SignatureDoesNotMatch。为 AWS 方案编写的客户端生成的签名可直接通过校验。

query 参数 edits

以下参数对任何请求类型都会叠加(并覆盖)路径中的 edits:

https://img.example.com/<base64>?width=400&height=300&fit=cover&format=webp&grayscale=true

支持:format、fit、width、height、 rotate、flip、flop、grayscale / greyscale。width/height 传 0 或空即清空该维度;flip/flop/grayscale 的 "0"、"false"、空串视为假。其余 query 参数在边缘就被丢弃(不会打散缓存键)。解析失败返回 400 QueryParameterParsingError。

常用编辑

缩放、格式、质量

{
  "key": "photos/cat.jpg",
  "edits": {
    "resize": { "width": 800, "fit": "inside" },
    "jpeg": { "quality": 75 }
  },
  "outputFormat": "jpeg"
}

fit 可取 cover、contain、fill、 inside(默认)、outside。宽高会四舍五入,≤ 0 返回 400 InvalidResizeException。输出 WebP 时可加顶层 "effort": 0–6(默认 4)。

智能裁剪(人脸检测)

{ "key": "people/team.jpg",
  "edits": { "smartCrop": { "faceIndex": 0, "padding": 40 } } }

由 Cloud Vision FACE_DETECTION 找出人脸;裁剪框为选中人脸的包围框向外扩 padding 像素,并收敛到图内。人脸按包围框面积从大到小排序,因此 faceIndex: 0 永远是最显著的人脸。未检测到人脸时返回整图。参数越界分别返回 400 SmartCrop::FaceIndexOutOfRange 与 400 SmartCrop::PaddingOutOfBounds。 "smartCrop": true 等价于全默认值。

内容审核

{ "key": "ugc/upload-123.jpg",
  "edits": { "contentModeration": {
      "minConfidence": 75, "blur": 50,
      "moderationLabels": ["Adult", "Violence"] } } }

Cloud Vision SafeSearch 的可能性等级会映射为 0–100 的置信度 (VERY_UNLIKELY=0 … VERY_LIKELY=100),再与 minConfidence(默认 75)比较。任一选中类别命中——类别为 Adult、Violence、 Racy、Medical、Spoof,同时兼容 "Explicit Nudity"、"Suggestive" 等 Rekognition 别名——图片就按给定 sigma 模糊(有效区间 0.3–1000)。省略 moderationLabels 表示任意类别命中即生效。

叠加水印(overlay)

{ "key": "photos/cat.jpg",
  "edits": {
    "resize": { "width": 800 },
    "overlayWith": {
      "bucket": "my-image-bucket",
      "key": "brand/logo.png",
      "alpha": "50",
      "wRatio": "20",
      "options": { "left": "-10", "top": "90p" }
    } } }
  • alpha —— 额外透明度 0–100(越界按 0 处理)。
  • wRatio/hRatio —— 把水印缩放到底图的百分比大小。
  • options.left/top —— 像素偏移;负值表示从右/下边缘反算;NNp 表示百分比(如 90p = 距顶 90%)。
  • 水印叠加失败返回 400 OverlayImageException。

圆形裁剪(round crop)

{ "key": "avatars/user.jpg",
  "edits": { "resize": { "width": 200, "height": 200, "fit": "cover" },
             "roundCrop": true } }

裁出椭圆(默认取内切圆/椭圆)并裁掉多余边缘。也可以传对象,用 rx、ry、top、left 控制椭圆。不带透明通道的输出格式(JPEG)外圈会呈黑色——配合 "outputFormat": "png" 或 webp 保留透明。

动图(GIF / WebP)

{ "key": "stickers/dance.gif",
  "edits": { "resize": { "width": 240 } },
  "outputFormat": "webp" }

GIF 源图自动逐帧处理(也可用 "edits": { "animated": true|false } 强制指定)。单帧图片自动回落到静态处理。动图会跳过 rotate、 smartCrop、roundCrop 与 contentModeration。

自定义响应头

{ "key": "docs/diagram.png",
  "headers": { "Cache-Control": "max-age=86400,public",
               "X-Robots-Tag": "noindex" } }

响应头先经过拒绝列表过滤(Set-Cookie、Authorization 及 x-amz-*/content-* 前缀等安全敏感头会被丢弃),见 API 参考。仅 Default 请求支持自定义响应头。

限时 URL(expires)

https://img.example.com/<base64>?expires=20261231T235959Z&signature=…

时间戳为严格的 UTC YYYYMMDDTHHmmssZ 格式。格式错误返回 400 ImageRequestExpiryFormat;已过期返回 400 ImageRequestExpired。校验通过后响应的 Cache-Control 被改写为 max-age=<剩余秒数>,public,确保 CDN 不会缓存超过有效期。请与签名配合使用——否则调用方把参数删掉即可绕过。

兜底图(fallback image)

配置 ENABLE_DEFAULT_FALLBACK_IMAGE=Yes 与 DEFAULT_FALLBACK_IMAGE_BUCKET/KEY 后,任何处理错误都会返回兜底图而不是错误 JSON——HTTP 状态码仍保留原始错误码,监控告警不受影响。Cache-Control 优先级:兜底对象自带元数据 → 请求 headers → max-age=31536000,public。

自动 WebP

开启 AUTO_WEBP=Yes 后,Accept 头包含 image/webp 的请求自动输出 WebP。优先级: edits.toFormat > AUTO_WEBP > JSON outputFormat。CDN 缓存键包含归一化后的 Accept 头,两种变体各自正确缓存。
