部署规划

成本

仅为估算。下表基于 2026 年 7 月 asia-southeast1 区域的公开牌价和文中假设粗略测算。实际账单取决于图片大小、缓存命中率、流量地域分布与商务折扣,请务必用 Google Cloud 价格计算器复核。

假设条件

  • 处理后图片平均大小:45 KB
  • Cloud CDN 缓存命中率:90%(只有未命中请求到达 Cloud Run)
  • Cloud CDN 出向流量(APAC 互联网):$0.09–0.14 / GB(随用量分档)
  • Cloud Run:1 vCPU / 1 GiB,每次未命中平均处理约 250 ms,min-instances=0
  • 负载均衡器:1 条转发规则约 $18/月,另有出向数据处理费约 $0.008–0.012/GB
  • 不含源图存储费用(客户自有桶)

降本抓手:提高缓存命中率(长 Cache-Control、稳定的 URL、归一化的 query——方案已内置)、开启 AUTO_WEBP 可减少 25–35% 出向字节;若不在意冷启动延迟,保持 min-instances=0。

安全

请求签名

设置 ENABLE_SIGNATURE=Yes 后,所有 URL 必须携带 HMAC-SHA256 签名。密钥以 JSON 文档形式存放在 Secret Manager,SECRET_KEY 指定取哪个键。待签串为 path 加排序后的 query,算法与 AWS 方案完全一致,现有 AWS 客户端生成的签名可直接通过校验。示例见 使用方案——签名请求。

源桶白名单

SOURCE_BUCKETS 是逗号分隔的白名单,第一个为默认桶。请求指定白名单以外的桶会被 403 ImageBucket::CannotAccessBucket 拒绝。只需给 Cloud Run 运行时服务账号 (service account)授予这些桶上的 roles/storage.objectViewer——本服务不需要任何写权限。

网络入站

Cloud Run 服务设置为 ingress=internal-and-cloud-load-balancing,只能通过负载均衡器(即经过 Cloud CDN)访问,直接请求 *.run.app 地址会被拒绝。

其他控制项

  • 响应头拒绝列表(deny list) —— Default 请求中的自定义响应头会先过一遍拒绝列表(authorization、set-cookie、x-amz-* 前缀等),详见 API 参考。
  • CORS —— 默认关闭;需要时设 CORS_ENABLED=Yes 并配置 CORS_ORIGIN。
  • 限时 URL —— expires 参数让过期链接返回 400 ImageRequestExpired。
  • Demo UI —— 仅供评估演示,不适用于生产环境;开启签名的部署不支持 Demo UI。

区域选择

参考部署将 Cloud Run 与各桶放在 asia-southeast1(新加坡)。Cloud CDN 与负载均衡器本身是全球性的,与后端区域无关;建议选择离源数据和未命中流量主体最近的区域。
