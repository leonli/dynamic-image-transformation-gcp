卸载方案

用与部署时相同的方式卸载。

Launch Wizard 部署

./infra/launch-wizard/launch-wizard.sh --destroy

向导会回放当初记录的变量值,展示销毁计划并要求确认,底层执行的是 terraform destroy。

Terraform 部署

cd infra/terraform
terraform destroy

确认前请仔细审阅销毁计划。

会删除的资源

  • Cloud Run 服务 dit-image-handler 及其 serverless NEG
  • 负载均衡器 dit-lb:转发规则、全局 IP、托管 SSL 证书、URL map、后端服务及 Cloud CDN 配置
  • 方案自建桶(…-dit-demo、…-dit-docs、…-dit-logs)及其内容
  • 密钥 dit-signature-secret(若由部署创建)
  • 部署创建的资源级 IAM 绑定

会保留的资源

  • 你的源图桶 —— 部署从不创建或删除 source_buckets 中列出的客户数据桶。
  • Artifact Registry(仓库 dit)中的容器镜像 —— 不需要时手动删除: 
    gcloud artifacts repositories delete dit --location=asia-southeast1
  • 已写入 Cloud Logging 的日志(按保留策略过期)。
  • 你自行创建的 DNS 记录 —— 图片域名的 A 记录请自行清理。

销毁之前:如果仍有客户端把图片域名解析到这台负载均衡器,销毁后会立即报错。请先改走或删除 DNS 记录,等 TTL 过期再执行销毁。

销毁中断后直接重跑即可——Terraform 会从已记录的状态续做。兜底手段:在控制台按前缀 dit- 搜索,确认上述资源均已清理。
