API 参考

请求类型

判定顺序(先匹配先生效):

  1. Default —— 路径为合法 base64 且解码为 JSON:GET /<base64(JSON)>
  2. Custom —— REWRITE_MATCH_PATTERN 与 REWRITE_SUBSTITUTION 均已配置;重写后按 Thumbor 解析
  3. Thumbor —— 路径无扩展名,或以 .jpg .jpeg .png .webp .tiff .tif .svg .gif .avif 结尾
  4. 否则 → 400 RequestTypeError

Default 请求 schema

字段类型必填说明
bucketstring否源桶。必须在 SOURCE_BUCKETS 白名单内(否则 403 ImageBucket::CannotAccessBucket)。缺省取白名单第一个。
keystring是源图对象 key。
editsobject否编辑管线(见下)。edits 与 outputFormat 都缺省时返回原始字节。
outputFormatstring否取值:jpg jpeg png webp tiff heif heic raw gif avif。
effortnumber否仅 WebP。截断为整数,有效区间 0–6,越界回落到 4。
headersobject否附加响应头,经拒绝列表过滤。

支持的 edits 键

分组键
通道(Channel)removeAlpha, ensureAlpha, extractChannel, joinChannel, bandbool
颜色(Color)tint, greyscale, grayscale, pipelineColourspace, pipelineColorspace, toColourspace, toColorspace
操作(Operation)rotate, flip, flop, affine, sharpen, median, blur, flatten, unflatten, gamma, negate, normalise, normalize, clahe, convolve, threshold, boolean, linear, recomb, modulate
格式(Format)jpeg, png, webp, gif, avif, tiff, heif, toFormat
缩放(Resize)resize, extend, extract, trim
方案特有overlayWith, smartCrop, roundCrop, contentModeration, crop, animated

管线顺序:先执行 resize(缺省注入 fit=inside),其余 edits 按对象键序执行。直通键一一映射到对应的 sharp 调用。

query 参数

参数作用
signaturepath + 排序后 query 的 HMAC-SHA256(hex),见签名。
expiresUTC YYYYMMDDTHHmmssZ;过期链接被拒绝。
format覆盖 toFormat。
fit覆盖 resize.fit。
width / height覆盖缩放尺寸;0 或空即清空该维度。
rotate覆盖旋转角度;空值表示自动转正。
flip / flop布尔;"0"、"false"、空串为假。
grayscale / greyscale布尔,假值规则同上。

其余 query 参数在入口中间件被剔除;重复参数取最后一个值;保留下来的参数按键排序(保证 CDN 缓存键稳定)。

错误码

错误以 JSON 返回:{"status": n, "code": "…", "message": "…"}, Content-Type: application/json,附带 CORS 头;Cache-Control 4xx 为 max-age=10,public、5xx 为 max-age=600,public。

状态码code触发场景
400RequestTypeError路径不匹配任何请求类型(扩展名不受支持)。
400DecodeRequest::CannotDecodeRequestbase64 路径无法解码为合法 JSON。
400DecodeRequest::CannotReadPath路径为空或不可读。
400QueryParameterParsingErrorquery 编辑参数解析失败。
400InvalidResizeException缩放宽/高取整后 ≤ 0。
400Crop::AreaOutOfBounds手工裁剪区域超出图片范围。
400OverlayImageException水印/叠加处理失败。
400SmartCrop::FaceIndexOutOfRangefaceIndex ≥ 检出的人脸数。
400SmartCrop::PaddingOutOfBoundspadding 使裁剪框超出图片。
400AuthorizationQueryParametersError要求签名但未携带(“Query-string requires the signature parameter.”)。
400ImageRequestExpiryFormatexpires 不符合 YYYYMMDDTHHmmssZ。
400ImageRequestExpiredexpires 已过期。
403ImageBucket::CannotAccessBucket请求的桶不在白名单内。
403SignatureDoesNotMatch签名校验失败。
404NoSuchKey“The image <key> does not exist or the request may not be base64 encoded properly.” 任何 GCS 读取失败都映射到此错误。
413TooLargeImageException仅在 COMPAT_AWS_LIMITS=Yes 时:处理结果 base64 编码后超 6 MB。
500SignatureValidationFailure密钥读取失败 / 签名校验过程中出现意外错误。
500SmartCrop::ErrorVision 人脸检测意外失败。
500Rekognition::DetectModerationLabelsErrorVision SafeSearch 调用失败(错误名保留 AWS 命名以兼容)。
500InternalError“Internal error. Please contact the system administrator.” —— 未归类的其他失败。

启用兜底图后,错误响应的响应体是兜底图,但状态码仍保留原错误码。

响应头

响应头取值
Cache-Control源对象的 Cache-Control,缺省 max-age=31536000,public;会被 expires(剩余秒数)与自定义 headers 覆盖。
Content-Type发生格式转换时为 image/<格式>;否则取源对象类型;仍无法确定时按魔数(magic number)推断(PNG/WebP/TIFF/GIF/JPEG/AVIF)。
Expires / Last-Modified从源对象元数据透传。
Access-Control-Allow-MethodsGET
Access-Control-Allow-HeadersContent-Type, Authorization
Access-Control-Allow-Credentialstrue
Access-Control-Allow-Origin仅 CORS_ENABLED=Yes 时输出 CORS_ORIGIN 的值。
AgeCloud CDN 缓存命中时附加(排查缓存问题的关键信号)。

响应头拒绝列表

自定义 headers 中命中以下条目(忽略大小写)的会被丢弃:

  • 精确匹配:authorization, connection, server, transfer-encoding, referrer-policy, permissions-policy, www-authenticate, proxy-authenticate, x-api-key, set-cookie
  • 前缀匹配:x-frame-*, x-content-*, x-xss-*, strict-transport-*, permissions-*, x-amz-*, x-amzn-*, access-control-*, cross-origin-*, content-*
